### Web 安全漏洞相关

#### 注入类
- LDAP 注入：攻击者通过构造恶意的 LDAP 查询语句，绕过身份验证或获取敏感信息。
- XML 注入：针对使用 XML 数据的应用程序，通过注入恶意 XML 代码来执行非法操作。
- XPath 注入：利用 XPath 表达式解析漏洞，构造恶意表达式获取或篡改 XML 数据。
- 邮件头注入：通过注入恶意邮件头信息，实现垃圾邮件发送、钓鱼攻击等。

#### 跨站相关
- DOM - Based XSS：基于文档对象模型（DOM）的跨站脚本攻击，通过修改页面的 DOM 结构来注入恶意脚本。
- 存储型 XSS：攻击者将恶意脚本存储在服务器端，当其他用户访问包含该脚本的页面时触发攻击。
- 反射型 XSS：攻击者将恶意脚本作为参数嵌入 URL 中，当用户点击该链接时，服务器将脚本反射回页面并执行。
- 跨站请求伪造（CSRF）变体
    - 基于 JSON 的 CSRF：针对使用 JSON 数据交互的应用程序的 CSRF 攻击。
    - 登录 CSRF：诱导用户在已登录状态下执行恶意操作，如修改密码、转账等。

#### 服务器配置与管理漏洞
- 服务器信息泄露：如服务器版本号、中间件信息、数据库类型等泄露，为攻击者提供攻击线索。
- 错误信息泄露：应用程序抛出详细的错误信息，可能包含敏感信息或代码逻辑，帮助攻击者进行漏洞分析。
- 目录索引泄露：服务器开启目录索引功能，导致敏感文件或目录被公开访问。
- 配置文件泄露：如 `.htaccess`、`config.php` 等配置文件泄露，可能包含数据库连接信息、密钥等。

#### 会话管理漏洞
- 会话固定攻击：攻击者通过诱导用户使用其预先获取的会话 ID 进行登录，从而控制用户会话。
- 会话超时设置不当：会话超时时间过长，增加了会话被劫持的风险；过短则影响用户体验。
- 会话 ID 预测：如果会话 ID 生成算法不安全，攻击者可能预测出有效的会话 ID，从而劫持会话。

#### 文件处理漏洞
- 文件上传漏洞绕过
    - 扩展名绕过：通过修改文件扩展名或使用特殊字符绕过上传文件类型检查。
    - 内容类型绕过：修改文件的 MIME 类型绕过服务器的文件类型验证。
    - 路径遍历绕过：利用路径遍历漏洞上传文件到非预期的目录。
- 文件包含漏洞变体
    - 远程文件包含（RFI）绕过：通过绕过服务器的远程文件包含限制，执行远程恶意脚本。
    - 本地文件包含（LFI）利用：利用本地文件包含漏洞读取敏感文件或执行任意代码。

### Web 应用架构与技术相关

#### 框架与库安全
- Django 安全问题
    - 模板注入：攻击者通过构造恶意模板代码，执行任意 Python 代码。
    - 表单验证漏洞：绕过 Django 表单验证机制，提交非法数据。
- Flask 安全问题
    - 路由注入：通过构造恶意路由参数，执行任意代码或获取敏感信息。
    - 会话管理漏洞：如会话 ID 泄露、会话固定等问题。
- jQuery 安全问题
    - XSS 风险：由于 jQuery 对 HTML 内容的操作，可能存在 XSS 漏洞。
    - 事件绑定漏洞：恶意脚本可能通过事件绑定机制执行。
- React 安全问题
    - 注入攻击：如属性注入、事件注入等，可能导致 XSS 或其他安全问题。
    - 状态管理漏洞：错误的状态管理可能导致数据泄露或代码执行。

#### API 安全
- API 密钥泄露：API 密钥被泄露后，攻击者可以使用该密钥访问受保护的 API 资源。
- API 暴力破解：攻击者通过不断尝试不同的请求参数，破解 API 的认证机制。
- API 数据篡改：攻击者修改 API 请求中的数据，执行非法操作。
- GraphQL 安全问题
    - 查询复杂度攻击：攻击者通过构造复杂的 GraphQL 查询，耗尽服务器资源。
    - 信息泄露：GraphQL 查询可能返回过多的敏感信息。

#### 单页应用（SPA）安全
- 前端路由安全：攻击者通过构造恶意的前端路由参数，执行任意代码或获取敏感信息。
- 状态管理安全：SPA 的状态管理机制可能存在漏洞，导致数据泄露或代码执行。
- 跨域资源共享（CORS）问题：不当的 CORS 配置可能导致跨域请求被恶意利用。

#### 微服务架构安全
- 服务间认证与授权：微服务之间的认证和授权机制不完善，可能导致服务被非法调用。
- 服务发现安全：服务发现机制可能被攻击，导致服务被错误路由或拒绝服务。
- 数据传输安全：微服务之间的数据传输可能被窃听或篡改，需要确保数据的保密性和完整性。

### Web 安全防护与检测相关

#### 防护机制
- 验证码安全
    - 验证码绕过：攻击者通过识别验证码或利用验证码生成算法的漏洞绕过验证码验证。
    - 验证码强度：验证码的复杂度不够，容易被自动化程序破解。
- 安全代理与网关
    - Web 应用防火墙（WAF）绕过：攻击者通过构造特殊的请求绕过 WAF 的防护。
    - API 网关安全：API 网关的配置不当可能导致安全漏洞，如认证绕过、数据泄露等。
- 内容安全策略（CSP）绕过
    - 通配符滥用：CSP 规则中使用过多的通配符，降低了 CSP 的安全性。
    - 绕过 CSP 限制：攻击者通过利用浏览器漏洞或 CSP 规则的漏洞，加载恶意脚本。

#### 检测技术
- 漏洞扫描器误报与漏报
    - 误报处理：漏洞扫描器可能将正常的应用程序行为误判为安全漏洞，需要进行人工验证和处理。
    - 漏报风险：由于扫描器的局限性，可能无法检测到某些新型或复杂的安全漏洞。
- 网络流量分析技术
    - 异常流量检测：通过分析网络流量的特征，检测异常的请求模式，如大量的请求、异常的请求频率等。
    - 恶意流量溯源：追踪恶意流量的来源，确定攻击者的位置和攻击路径。
- 日志分析与审计
    - 日志篡改检测：检测日志文件是否被篡改，确保日志的完整性和可靠性。
    - 异常行为审计：通过分析日志记录，发现用户的异常行为，如异常的登录时间、异常的操作记录等。
